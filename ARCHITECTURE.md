## 数据流（以"生成综述"为例）

```
用户勾选文献 → 点击"生成综述"
    ↓
前端发送 POST /api/ask/stream
    ↓
query.py 验证JWT → 检查每日限额 → 调用 retriever.ask()
    ↓
retriever 检测到"综述"关键词
    ↓
每篇文献：
  ├─ 抓首页3块（提取作者/标题/期刊）
  └─ 按章节抓12块（摘要/引言/方法/结果/结论）
    ↓
拼装 Prompt（正文内容 + 首页元信息 + APA格式要求）
    ↓
DeepSeek 流式输出 → SSE 推送到浏览器
    ↓
前端逐字渲染 → 完成后保存历史
```

---

## 三种问答模式

| 模式 | 触发条件 | 每篇块数 | 说明 |
|------|---------|---------|------|
| 普通问答 | 默认 | 8块(全库) | 语义检索最相关内容 |
| 多文档对比 | 选中≥2篇 + 含"分析/对比/所有"等词 | 4块/篇 | 逐篇分析 + 综合对比 |
| 学术综述 | 含"综述/文献综述/写一篇"等词 | 12块/篇 + 首页3块 | 五段式学术格式 + APA参考文献 |

## 文件结构

```
kb_project/
├── main.py              # FastAPI 入口，路由注册
├── config.py            # 全局配置/环境变量
├── requirements.txt     # Python 依赖
├── Procfile             # Railway 启动命令
├── railway.json         # Railway 构建配置
├── runtime.txt          # Python 版本声明
├── db/
│   └── database.py      # User / Document 数据模型 (SQLite)
├── routers/
│   ├── auth.py          # 注册/登录/JWT/管理员激活
│   ├── upload.py        # PDF上传/文档列表/删除/限额
│   ├── query.py         # 问答/综述/章节/统计
│   └── payment.py       # 支付预留接口
├── services/
│   ├── pdf_processor.py # PDF解析/章节识别/中英文切片
│   ├── embedder.py      # Jina API向量化/Chroma读写/首页提取
│   └── retriever.py     # 三种检索模式/Prompt构建/LLM调用
└── frontend/
    └── index.html       # 单文件前端（Landing Page + App）
```

## 技术栈

| 层次 | 技术 |
|------|------|
| 前端 | 纯 HTML/CSS/JS，单文件，无框架 |
| 后端 | Python 3.10 + FastAPI + Uvicorn |
| 认证 | JWT (PyJWT) + SHA256 密码哈希 |
| PDF解析 | PyMuPDF + MarkItDown |
| 向量化 | Jina AI Embeddings API (jina-embeddings-v3, 768维) |
| 向量数据库 | Chroma (本地持久化) |
| 关系数据库 | SQLite (SQLAlchemy ORM) |
| LLM | DeepSeek API (deepseek-chat) |
| 部署 | Railway (免费套餐, Nixpacks构建) |
| 版本控制 | GitHub (topsaicp/docmind) |
