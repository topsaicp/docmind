工作原理

三步构建私人知识库

1
上传 PDF 文献
支持英文、中文学术论文及任意文字型 PDF。系统自动检测语言，英文论文用 PyMuPDF 精准解析章节结构，中文文档用 MarkItDown 提取全文，自动去除参考文献噪音。
2
语义向量化入库
文本按章节切分后，由 Jina AI 向量化 API 转化为高维语义向量，存入 Chroma 向量数据库。支持中英双语语义检索，章节名采用「Abstract（摘要）」双语格式存储。
3
精准检索 + AI 作答
提问时检索最相关文本块，由 DeepSeek 主力模型生成有据可查的回答并标注来源章节。遇到图表可框选截图，Gemini 视觉模型实时解读图表与公式内容。
核心功能

为学术研究而生

📖
PDF 精读对话
三栏布局：左侧章节目录 + 中栏 PDF 原文阅读 + 右栏 AI 对话。可框选页面任意区域截图，由 Gemini 视觉模型实时解读图表、公式和表格，也可粘贴/上传图片分析。

多文献知识库问答
同时勾选多篇文献提问，系统对每篇单独检索代表性内容，输出逐篇分析报告及综合对比。支持按章节定向提问（摘要、方法、结论），可开启联网检索补充最新信息。

文献综述生成

论文章节撰写
提供摘要、引言、文献综述、研究方法、实验结果、讨论与结论等标准章节框架，结合已上传文献内容逐章 AI 辅助撰写，支持一键生成全文并导出。

英文降率工具

常见问题

您可能想了解的

PDF 文件存储在服务器，不会公开。处理时文本片段会发送给以下第三方 API：Jina AI（向量化）、DeepSeek（问答/综述/撰写）、Gemini（图表截图分析）。原始 PDF 文件不会发送给任何 AI 服务，仅发送提取后的文本片段。
支持中英文学术论文、报告、书籍等文字型 PDF，单文件最大 50MB。系统自动检测语言，英文论文使用 PyMuPDF 精准解析章节结构，中文文档使用 MarkItDown 提取。纯扫描件无文字层的效果较差。
进入精读页面后，点击顶部「✂️ 框选分析」按钮，鼠标变为十字形，在 PDF 上拖动选择任意区域（图表、公式、表格均可），松开后右侧对话框出现截图预览，可直接发送或输入追问内容，Gemini 视觉模型将实时分析图片内容。
选中多篇文献时，系统对每篇单独检索代表性内容块（优先摘要、引言、结论），再合并给 AI 逐篇分析并输出综合对比。不会出现某一篇独占全部回答的情况。
当前 AI 自动降率仅支持英文（.txt / .docx）。中文降率可选择「人工服务」，发送文件至 topsai@protonmail.com 获取报价。退款申请：AI 自动模式达到退款条件（AI 率超 20% 或重复率超 15%），发邮件附 Turnitin 报告截图，48 小时内处理。
发送邮件至 topsai@protonmail.com，注明「取消订阅」及注册邮箱，24 小时内处理并确认。

总则

DocMind（以下简称"我们"）重视您的隐私。本隐私政策说明我们在您使用 DocMind 平台（https://docmindai.up.railway.app/）时收集、使用和保护您的信息的方式。使用本服务即表示您同意本政策的条款。

我们收集的信息

  • 账户信息：注册时的邮箱地址及加密后的密码（SHA-256 + 随机盐哈希存储，原始密码不可恢复）。
  • 上传内容：您上传的 PDF 文件。文件内容经提取后以向量索引形式存储，用于 AI 检索服务。原始文件暂存于服务器，删除文档时同步清除。
  • 使用数据：每日问答次数、文档数量，用于执行套餐限额。
  • 问答历史：存储于您本地浏览器的 localStorage，不上传至服务器，清除浏览器数据即可删除。

信息使用目的

  • 提供 AI 知识库问答、文献综述、论文撰写及降率处理服务
  • 执行免费/付费套餐限额及计费管理
  • 向您发送服务相关通知（如重大政策变更）
  • 分析匿名使用数据以改进产品

数据存储与安全

  • 账户数据库：PostgreSQL，托管于 Supabase（节点位于美国西部）
  • 向量索引：Chroma 向量数据库，存储于 Railway 服务器（美国）
  • 问答历史：仅存于用户本地浏览器（localStorage），服务器不保留
  • 传输安全：所有 API 通信均通过 HTTPS/TLS 加密
  • 密码安全：使用 SHA-256 + 随机盐哈希存储，平台无法还原明文密码

第三方服务

为提供服务，我们使用以下第三方工具，它们可能处理您的部分数据：

  • DeepSeek AI（北京深度求索人工智能基础技术研究有限公司）— 提供大语言模型推理服务；您的文档检索内容和问题会发送至其 API
  • Jina AI（德国柏林）— 提供文本向量化（Embedding）服务
  • Railway（美国）— 应用服务器托管
  • Supabase（美国/新加坡）— PostgreSQL 数据库托管

本地存储（Cookie 替代）

我们使用浏览器 localStorage 存储登录凭证（JWT Token），用于保持登录状态，不设置第三方追踪 Cookie，也不使用任何广告追踪技术。
